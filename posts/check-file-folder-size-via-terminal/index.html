使用命令行检查文件/文件夹大小

April 22, 2025
til

背景：我使用 macOS。

在命令行工具里，有很多工具可以知道文件大小:

ls -la 一般情况下使用，可以直观的文件大小，缺点是不知道文件夹大小 du * 会列出当前目录下所有文件夹内，所有文件的大小 du -hxd1 会列出当前目录下，所有子文件夹的的大小 gdu 用 golang 写的，支持多线程，作用和 du * 一样，有点是性能好，速度快

不过我自己偏好 ncdu，有可视化界面，会显示子文件/子文件夹占用的空间百分比，还可以直接简单操作：

ncdu 执行速度没有 gdu 快——但最新版本提供了多线程的实验特性，输入 ncdu -t8 即可调用，速度要快上不少。

PS：谢谢「Emacs 中文」 里各位认真有趣的群友。
